üìä Research Area

Stakeholder Interviews

I conducted interviews with 4 internal stakeholders across different teams to uncover usability concerns with the current enterprise tools. Each conversation highlighted inefficiencies in cross-functional communication, manual tracking workflows, and poor task visibility.

These insights formed the foundation for mapping the problem space and prioritizing design decisions around discoverability, speed, and workflow clarity.

Workflow Mapping

I analyzed 3 core workflows used by internal operations teams, diagramming each interaction step. Pain points included repetitive status checks, disconnected tools, and lack of transparency on ownership or task priority.

Mapping these flows helped me identify opportunities for UI consolidation, better labeling, and automation triggers.

Key Themes

  • Visibility: Users struggled to find key task data without navigating multiple tools.
  • Speed: Manual processes delayed communication and slowed decision-making.
  • Confidence: Participants lacked clarity on who owned what and when updates were expected.

üé® Design Area

Design Principles

I framed three core principles to guide every decision:

  • Reduce Friction: Streamline repetitive actions with automation and clear pathways.
  • Clarity First: Prioritize hierarchy and legibility to improve cross-functional task visibility.
  • Empower Ownership: Provide role-based views that support timely decisions and accountability.

User Flows

I developed optimized flows for three roles: operators, analysts, and team leads. Each flow aimed to minimize steps between task discovery and resolution, while enhancing role-specific visibility.

  • Operators: Quick task execution and submission confirmation
  • Analysts: Filter-heavy dashboards for scanning patterns and tagging priorities
  • Team Leads: Oversight view with escalation flags and historical insights

Wireframes & Explorations

I began with grayscale layouts that prioritized information density and modularity. I explored different ways to surface next steps, show real-time task ownership, and reduce scan time.

These explorations led to a responsive panel-based layout that emphasized vertical reading flow, repeat patterns, and collapsible metadata.

High-Fidelity UI

I polished the final interface with Amazon‚Äôs design system and accessible interaction patterns. Dark mode support, clear status indicators, and bulk-action feedback helped streamline team operations.

Custom filtering, inline guidance, and visible progress states gave users more control ‚Äî and fewer chances to feel lost in the system.

Usability Testing Approach

I collaborated with UX researchers and internal operators to run two feedback loops ‚Äî one using a clickable Figma prototype, and another through a sandbox testing environment.

Participants were asked to complete tasks like finding high-priority issues, assigning them, and submitting resolutions. I observed hesitation, misclicks, and areas where system status wasn‚Äôt clear.

Key Feedback

  • Operators needed confirmation after submitting ‚Äî visual feedback was too subtle.
  • Filters worked well, but some icons were unclear without labels.
  • Escalation logic wasn‚Äôt intuitive ‚Äî users wanted clarity around what gets flagged and why.

These insights helped refine interaction patterns and improve task confidence across roles.

What I Changed

  • Added toast confirmations and icon animations for key task actions.
  • Implemented optional labels for key UI icons in mobile view.
  • Updated escalation logic to show tooltips and rationale for status transitions.

These changes led to reduced drop-off, quicker onboarding, and improved team satisfaction.
