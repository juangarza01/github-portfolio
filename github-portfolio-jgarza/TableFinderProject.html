Design Principles

From the start, I knew TableFinder had to be fast, mobile-first, and inclusive. I set three principles:

  • Speed-First: Users should be able to view open tables and book within seconds ‚Äî with no distractions.
  • Minimalist Navigation: Focused flows that remove choice overload and streamline reservations.
  • Accessible Clarity: Use of strong color contrast, legible type, and clear states for all modes and users.

User Flows

I sketched primary flows around three key needs:

  • Find a Table: Input time/date ‚Üí see options ‚Üí book.
  • Repeat Booking: Re-book favorites with a single tap.

Sketches & Wireframes

I began with sketches focused on interaction speed ‚Äî removing non-essentials and simplifying the table selection view. Early wireframes leaned on modular components for mobile optimization, like tappable cards and grid selectors.

I refined the interface with color-coded statuses, large CTA buttons, and a card-based layout for hosts. Elements like animated checkmarks, subtle microcopy, and tap-friendly UI affordances enhanced the experience without clutter.

üß™ Testing & Feedback

Usability Testing Approach

I conducted two testing rounds with 6 participants ‚Äî 3 diners and 3 restaurant hosts. They explored key flows using mid- and high-fidelity Figma prototypes on mobile devices.

Tasks included finding a table, making edits, toggling between guest/host modes, and confirming bookings. I tracked hesitations, errors, and completion speed to identify friction points.

Key Feedback

  • One-tap availability was loved, but users wanted visual confirmation (like animations or checkmarks).
  • Hosts asked for better labeling of walk-ins vs. reserved tables.
  • The calendar grid was preferred over a list view ‚Äî especially for visualizing a full week.

These insights confirmed both guest-facing and staff-facing needs ‚Äî and helped prioritize small but meaningful UI improvements.

What I Changed

  • Integrated animated feedback for success states (e.g. checkmarks, toasts).
  • Refined table status labels and added icon distinctions for host dashboard.
  • Upgraded calendar to a grid layout with clear day/time hierarchy and tappable areas.

These changes reduced cognitive load, improved confidence, and enhanced usability across devices.
